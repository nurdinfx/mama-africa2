# Environment Variables Setup

## Quick Start

1. Copy `.env.example` to `.env`:
   ```bash
   cp .env.example .env
   ```

2. Update the values in `.env` if needed (default values should work for local development)

3. Restart the development server to apply changes

## Available Environment Variables

### API Configuration
- `VITE_API_URL` - Full API URL (default: http://localhost:5000/api/v1)
- `VITE_API_BASE_URL` - Base backend URL (default: http://localhost:5000)
- `VITE_BACKEND_URL` - Backend server URL (default: http://localhost:5000)
- `VITE_SOCKET_URL` - WebSocket/Socket.io URL (default: http://localhost:5000)

### App Configuration
- `VITE_APP_NAME` - Application name (default: Mama Africa Restaurant)
- `VITE_APP_VERSION` - Application version (default: 1.0.0)
- `VITE_NODE_ENV` - Environment mode (development/production)

### Upload Configuration
- `VITE_MAX_FILE_SIZE` - Maximum file upload size in bytes (default: 10485760 = 10MB)
- `VITE_ALLOWED_FILE_TYPES` - Comma-separated list of allowed MIME types

### Feature Flags
- `VITE_ENABLE_DEMO_MODE` - Enable demo accounts (default: true)
- `VITE_ENABLE_SOCKET` - Enable real-time socket connections (default: true)
- `VITE_ENABLE_NOTIFICATIONS` - Enable notifications (default: true)

### Localization
- `VITE_TIMEZONE` - Application timezone (default: Africa/Mogadishu)
- `VITE_DATE_FORMAT` - Date display format (default: DD/MM/YYYY)
- `VITE_TIME_FORMAT` - Time display format (default: HH:mm:ss)
- `VITE_CURRENCY` - Currency code (default: USD)
- `VITE_CURRENCY_SYMBOL` - Currency symbol (default: $)

### Debug
- `VITE_DEBUG` - Enable debug mode (default: true in development)

## Important Notes

1. **Vite Prefix**: All environment variables must start with `VITE_` to be exposed to the client
2. **Restart Required**: Changes to `.env` require restarting the dev server
3. **Security**: Never commit `.env` file to version control (it's in .gitignore)
4. **Production**: Update values appropriately for production deployment

## Production Deployment

For production, create a `.env.production` file or set environment variables in your hosting platform:

```env
VITE_API_URL=https://your-api-domain.com/api/v1
VITE_BACKEND_URL=https://your-api-domain.com
VITE_SOCKET_URL=https://your-api-domain.com
VITE_NODE_ENV=production
VITE_DEBUG=false
```

## Troubleshooting

### Environment variables not working?
1. Make sure variable names start with `VITE_`
2. Restart the development server
3. Clear browser cache
4. Check console for any errors

### API connection issues?
1. Verify backend is running on the correct port
2. Check `VITE_API_URL` matches your backend URL
3. Ensure CORS is properly configured on backend
